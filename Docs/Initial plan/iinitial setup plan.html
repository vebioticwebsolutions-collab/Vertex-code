Key principle: The frontend team calls the API and displays what it returns. They never handle calculation logic, database queries, PDF generation, or email sending — all of that lives in the backend. The API is the contract between both sides.

Calculator Flow — Frontend Responsibilities
StepFrontend doesAPI callBackend does (invisible)
User picks type, enters specsShow relevant input fields · validate client-sidePOST /api/public/walkway/calculateChecks cache → runs calculation → stores result temporarily → returns quote_token
Lead capture formCollect name, location, phone, emailPOST /api/public/lead/submit (with quote_token)Saves lead details → sends OTP email → returns {success: true}
OTP entryShow 6-digit input · handle resend · countdown timerPOST /api/public/lead/verify-otpVerifies OTP → generates PDF → uploads to R2 → sends email → returns final result
Result displayShow rate, weight, specs from API response · show PDF linkNo call — result comes from verify-otp responseNothing more needed
Astro Islands note: The calculator (Steps 1–4) needs client-side interactivity (form state, OTP timer, conditional rendering). Use an Astro Island for this component — the rest of the page (header, SEO metadata, product info) can be fully static Astro.

Auth Flow
ActionFrontendAPI
LoginPOST credentials → store JWT token → redirect to /dashboardPOST /auth/login → returns {access_token, expires_in}
Every protected API callInclude Authorization: Bearer {token} headerBackend validates JWT · Returns 401 if expired
Token expiredCatch 401 → redirect to /login · clear stored tokenReturns {detail: "Token expired"}
LogoutClear token · redirect to /loginNo API call needed (stateless JWT)
Public Endpoints — no auth required
POST /api/public/walkway/calculate No auth
Request Body
// example: 2-fold without hook { "type": "2fold-no-hook", "length_mm": 2000, "width_mm": 300, "thickness_mm": 1.8 }
Response
{ "quote_token": "qt_abc123xyz", "status": "calculated" } // result held until OTP verified // do NOT show rate yet
POST /api/public/lead/submit No auth
Request Body
{ "quote_token": "qt_abc123xyz", "name": "Rahul Sharma", "location": "Mumbai, Maharashtra", "phone": "9876543210", "email": "rahul@example.com" }
Response
{ "status": "otp_sent", "message": "OTP sent to rahul@ex...", "expires_in_seconds": 600 }
POST /api/public/lead/verify-otp No auth
Request Body

Response (success)

Frontend team note: The frontend never queries the database directly. All data access goes through the API. The database is fully managed by the backend — the frontend team does not need Supabase credentials or any DB access.
